<objective>
Build the two pieces of pure infrastructure the email channel needs before any IMAP/SMTP wiring:
(1) an AES-256-GCM `secret-box` encrypt/decrypt helper — the codebase's first at-rest encryption
primitive, which Phase 4's LLM keys will reuse — and (2) a typed email-settings module over the
existing key/value `Setting` model, which encrypts credential fields on the way in and decrypts
them on the way out.

Purpose: Downstream plans (outbound send, inbound poll, settings UI) all read decrypted IMAP/SMTP
config through one module and never handle raw credentials or plaintext storage themselves (D-07,
SECURITY.md non-negotiable).
Output: `secret-box.ts` (+ unit tests), `channels/email/settings.ts`, and a documented
`APP_ENCRYPTION_KEY` in `.env.example`.

This plan is fully independent of 03-01 (it touches neither package.json nor the new schema fields —
`Setting` already exists) and runs in parallel with it in Wave 1.
</objective>

    - decryptSecret(blob) on a byte-flipped/tampered blob THROWS (GCM auth-tag failure) — never returns a value.
    - getKey path: with APP_ENCRYPTION_KEY unset, encryptSecret throws "APP_ENCRYPTION_KEY is required…"; with a key that decodes to != 32 bytes, it throws the "must decode to exactly 32 bytes" error.

    Create `src/lib/crypto/secret-box.ts` exactly as in 03-RESEARCH.md Code Examples: `ALGORITHM =
    "aes-256-gcm"`, `IV_LENGTH = 12`, `TAG_LENGTH = 16`; `getKey()` reads `process.env.APP_ENCRYPTION_KEY`,
    base64-decodes it, and throws if missing or not 32 bytes; `encryptSecret(plaintext)` generates a fresh
    `randomBytes(12)` IV, uses `createCipheriv("aes-256-gcm", key, iv)`, calls `cipher.getAuthTag()` after
    `final()`, and returns `Buffer.concat([iv, authTag, ciphertext]).toString("base64")`;
    `decryptSecret(packed)` slices iv(0..12)/authTag(12..28)/ciphertext(28..), calls
    `decipher.setAuthTag(authTag)` BEFORE `final()`. Use `node:crypto` built-ins only (no new dependency).

    Use no internal project imports (node:crypto only) so this file is safely bundleable by both Next.js
    (webpack) and the worker (esbuild).

    `process.env.APP_ENCRYPTION_KEY = randomBytes(32).toString("base64")`. Cover the four behaviors above.

    assert `decryptSecret` throws. For the key-validation tests, temporarily unset / set a short key and
