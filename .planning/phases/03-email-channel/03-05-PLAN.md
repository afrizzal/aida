Deliver agents' public replies by SMTP: build the SMTP transport factory + MIME composer, an
app-side pg-boss singleton so the messages Route Handler can enqueue a one-off send, the worker
send handler (with delivery-status transitions), and the thread-message "Failed to send — Retry"
affordance.

surfaced not silent). This is the codebase's FIRST app-side pg-boss enqueue (Phases 1–2 only ever
had the worker schedule its own recurring jobs).
Output: smtp-client, compose-outbound, boss-client, the send handler, messages-route enqueue,
delivery-status UI + retry.

`email-outbound-send` work handler is deliberately owned by plan 04 (which owns `worker/index.ts`
and imports `emailOutboundSendHandler` from this plan) so the two plans never both edit the worker
entrypoint.

    `export async function emailOutboundSendHandler(data: { messageId: string }): Promise<void>`:
    1. Load the message with its ticket + ticket.contact + org name:
       `prisma.message.findUnique({ where: { id: data.messageId }, include: { ticket: { include: { contact: true, organization: true } } } })`.
    2. If not found, or `direction !== "OUTBOUND"`, or `visibility !== "PUBLIC"`, or no `ticket.contact.email` → return (nothing to send).
    3. Build a scoped db `const db = scopedDb(message.organizationId);` and `const settings = await getEmailSettings(db);`. If `!settings.enabled` → return.
    4. Determine the outbound Message-ID: reuse `message.emailMessageId` if already set (retry re-send), else
       `buildOutboundMessageId(domainFromFromAddress)` where domain = the part after `@` in `settings.fromAddress`; persist it on the row (`db.message.update`) so retries + threading stay stable.
    5. Threading headers (D-23): query prior email Message-IDs for this ticket ordered by createdAt asc:
       `db.message.findMany({ where: { ticketId: message.ticketId, emailMessageId: { not: null } }, orderBy: { createdAt: "asc" }, select: { emailMessageId: true, direction: true } })`.
       `references` = those ids (mapped, non-null); `inReplyTo` = the LAST INBOUND email Message-ID (latest inbound), or undefined.
    6. `const transporter = createSmtpTransport(settings);` then
       `await transporter.sendMail(composeMail({ fromAddress: settings.fromAddress, fromName: message.ticket.organization.name, to: message.ticket.contact.email, subject: \`Re: ${message.ticket.subject} [#${message.ticket.number}]\`, bodyMarkdown: message.bodyMarkdown, messageId, inReplyTo, references }));`
    7. On success: `db.message.update({ where: { id: message.id }, data: { deliveryStatus: "SENT" } })`.
    8. On thrown error: `db.message.update({ where: { id: message.id }, data: { deliveryStatus: "FAILED" } })` then RETHROW (so pg-boss retries per the queue's retryLimit; after the final attempt the row stays FAILED for the UI). Never log `settings.smtpPassword` or full email bodies.
