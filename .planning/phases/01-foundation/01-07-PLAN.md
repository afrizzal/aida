       - base: FROM node:22-alpine AS base; RUN corepack enable.

         RUN pnpm exec esbuild src/lib/worker/index.ts --bundle --platform=node --format=cjs --target=node22 --outfile=dist/worker.cjs

         COPY --from=builder /app/public ./public
         COPY --from=builder /app/.next/standalone ./
         COPY --from=builder /app/.next/static ./.next/static
         COPY --from=builder /app/dist ./dist
         COPY --from=builder /app/prisma ./prisma
         EXPOSE 3000
         CMD ["node", "server.js"]   # app default; worker overrides command in compose
       (Do NOT copy a full app node_modules — standalone + the worker bundle are self-contained per Pitfall 4. The prisma/ folder is copied so the migrate step in Task 2 can run migrate deploy.)

       - migrate (one-shot): build: .; command runs `pnpm prisma migrate deploy` (needs the prisma CLI — use an image/stage that retains it, e.g. run the builder stage target, or `image: node:22-alpine` mounting nothing and `npx prisma migrate deploy` with prisma + schema available; simplest: target the builder stage via `build: { context: ., target: builder }` and command `pnpm prisma migrate deploy`); environment DATABASE_URL=postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@db:5432/${POSTGRES_DB}; depends_on db (service_healthy); restart: "no".
       - app: build: .; command ["node","server.js"]; depends_on { db: service_healthy, migrate: service_completed_successfully }; environment DATABASE_URL (host db), BETTER_AUTH_SECRET, BETTER_AUTH_URL, NEXT_PUBLIC_APP_URL, ADMIN_EMAIL/PASSWORD/NAME (passthrough for D-08), HOSTNAME=0.0.0.0, DB_POOL_MAX=10; expose 3000 (NO host port — Caddy fronts it); healthcheck using wget: ["CMD-SHELL","wget -qO- http://localhost:3000/api/health || exit 1"] interval 10s timeout 5s retries 5; restart unless-stopped; deploy.resources.limits.memory 512m.
       - worker: build: .; command ["node","dist/worker.cjs"]; depends_on { db: service_healthy, migrate: service_completed_successfully }; same DATABASE_URL; environment DB_POOL_MAX=5; restart unless-stopped; deploy.resources.limits.memory 1g.
       - caddy: image caddy:2; ports "80:80" and "443:443"; volumes ./Caddyfile:/etc/caddy/Caddyfile:ro, caddy_data:/data, caddy_config:/config; environment DOMAIN=${DOMAIN:-localhost}; depends_on app; restart unless-stopped.
       - volumes: postgres_data, caddy_data, caddy_config.

    2. Pool sizing (D-09): read DB_POOL_MAX in src/lib/db.ts (new pg.Pool({ connectionString, max: Number(process.env.DB_POOL_MAX) || 10 })) so app=10 + worker=5 = 15 < Postgres default 100. Add a documented DB_POOL_MAX line to .env.example.

       ```
       {$DOMAIN:localhost} {
         reverse_proxy app:3000 {
           header_up X-Forwarded-Proto {scheme}
           header_up X-Forwarded-Host {host}
         }
         encode zstd gzip
       }
       ```
       (Validate {$DOMAIN:localhost} against caddyserver.com docs — RESEARCH flagged MEDIUM confidence. localhost yields Caddy's internal-CA TLS; a real DOMAIN triggers Let's Encrypt auto-HTTPS.)
