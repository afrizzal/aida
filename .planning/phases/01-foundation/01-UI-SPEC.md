## Interaction Contracts

### Login Page

- **Layout:** centered card, max-width 400px, vertically centered in viewport, `min-h-screen` with `flex items-center justify-center`
- **Fields:** email (type=email, autocomplete=email), password (type=password, autocomplete=current-password)
- **Validation:** client-side zod validation on blur; server-side error on submit
- **Loading state:** submit button shows spinner (`animate-spin` Lucide `Loader2`) and is `disabled` during in-flight request
- **Error display:** red-bordered input field + error message below in destructive color, not a toast (inline errors for auth)
- **Password visibility:** no toggle required in Phase 1 (keep it simple)
- **No:** "Forgot password?", "Create account", or social login links â€” AIDA is admin-invite-only

### Setup Wizard

- **Layout:** centered card, max-width 400px, same centering as login
- **Steps:** single-page form (not multi-step tabs) â€” org name, org slug (auto-derived from name, editable), admin name, admin email, admin password, confirm password
- **Slug auto-derive:** on org name input, auto-populate slug field as `name.toLowerCase().replace(/[^a-z0-9]+/g, '-')` with debounce 300ms
- **Loading state:** same spinner pattern as login button
- **Error display:** inline per field; generic server error shown as a `sonner` toast
- **Redirect on success:** 302 to `/login` with a query param `?setup=complete` that shows a success toast on the login page

### App Shell â€” Sidebar

- **Width:** 240px fixed, not collapsible in Phase 1 (collapsible is Phase 2+)
- **Background:** `--muted` (secondary surface)
- **Logo area:** "AIDA" wordmark (text, not image) at 18px / weight 700, top of sidebar, 16px padding
- **Nav items:** Tickets, Knowledge Base, Settings â€” each 36px tall, 8px vertical padding, 12px horizontal padding, 14px body text
- **Active state:** `--primary` background, `--primary-foreground` text
- **Hover state:** `--accent` background (distinct from active)
- **Nav item icons:** Lucide `Inbox` (Tickets), `BookOpen` (Knowledge Base), `Settings` (Settings) â€” 16px, 4px gap before label
- **User area:** bottom of sidebar, avatar + name + email (truncated), 16px padding

### App Shell â€” Top Bar

- **Height:** 56px fixed
- **Background:** `--background` (matches main canvas, creates unified header)
- **Border:** 1px `--border` on bottom
- **Left:** page title (current section name, 18px / 600 weight)
- **Right:** theme toggle icon button (Lucide `Sun`/`Moon`, 20px) + user dropdown trigger
- **Theme toggle:** calls `setTheme()` from next-themes; tooltip shows "Light mode" / "Dark mode"

### Settings â€” AI Toggle

- **Section heading:** "AI Features" at 18px / 600 weight
- **Toggle row:** `Switch` component with label to the right "Enable AI" and description below at 12px muted "Allow AIDA to triage tickets and draft replies. Configure your AI provider in a future step."
- **Default state:** `aiEnabled: false` â€” switch is off
- **Persistence:** Server Action â†’ updates `setting` row in database (`key: "aiEnabled", value: "false"/"true"`)
- **No confirmation dialog** â€” toggling on is not destructive; toggle off is not destructive in Phase 1 (no AI code exists yet)

## Implementation Notes for Executor

1. **shadcn init first:** Run `npx shadcn@latest init --style new-york --base-color zinc --css-variables` as the first step in Wave 0. Verify `components.json` exists before proceeding.

2. **Tailwind v4 â€” no config file:** All design tokens live in `src/app/globals.css` under `@theme inline`. shadcn's init will write the CSS variable block. Do not create `tailwind.config.js` or `tailwind.config.ts`.

3. **Font loading:** Add Inter via `next/font/google` in `src/app/layout.tsx`. Apply `inter.variable` to `<html>` className. Set `--font-inter` via CSS variable. Use `font-family: var(--font-inter), system-ui, sans-serif` in globals.

4. **Theme provider placement:** Wrap `{children}` in `src/app/layout.tsx` with `<ThemeProvider attribute="class" defaultTheme="system" enableSystem>` from next-themes. This wires dark mode for all phases from day one.

5. **Sidebar background:** Use `bg-muted` (shadcn utility) which maps to `--muted`. Do NOT use a custom color; keep all colors in the CSS variable system.

6. **Active nav state:** Use `bg-primary text-primary-foreground` for the active nav item. Use `hover:bg-accent hover:text-accent-foreground` for hover. These are shadcn's own utility classes.

7. **Stub pages:** Empty state should be centered in the content area with the icon above heading. Lucide icon at 48px (`h-12 w-12`), heading at 18px/600, body at 14px/400/muted.

8. **Lucide icons for nav:** `Inbox` (Tickets), `BookOpen` (Knowledge Base), `Settings2` (Settings). Size: `h-4 w-4` (16px). Gap to label: `gap-2` (8px).
