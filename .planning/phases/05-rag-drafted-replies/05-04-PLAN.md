<objective>
Build the retrieval + grounded-drafting core: an org-scoped brute-force KNN query over `KbChunk`, a zod `DraftResultSchema`, a generalized prompt-injection fence for BOTH the ticket message and each retrieved KB chunk, a draft system prompt, and the `generateDraftReply(ticketId)` orchestrator that embeds the query, retrieves, applies a code-level groundedness gate (Success Criterion 4), and — only when relevant content exists — calls the existing `complete()` port and records a `DRAFT_GENERATED` audit event. Also fix Pitfall 3 by threading an optional `maxOutputTokens` through the LLM port so multi-paragraph drafts aren't truncated by Anthropic's hardcoded 1024 cap.

Purpose: This is the grounded-draft engine behind AIDA-16 and the no-hallucination guarantee (Success Criterion 4). 05-07's ticket UI calls the `generateDraftReply` Server Action this plan exposes.
Output: `src/lib/rag/` draft engine + LLM maxOutputTokens plumbing + the Server Action + an integration test proving groundedness + injection fencing + the audit row.

Scope note (Decision 1): v1 retrieval corpus is KB articles ONLY. AIDA-16's "KB/past-ticket context" is satisfied KB-only for v1; past-ticket embedding is deferred to Phase 6 (AIDA Insight, which already needs ticket-content analysis). This deferral is recorded here for honest requirement coverage.
