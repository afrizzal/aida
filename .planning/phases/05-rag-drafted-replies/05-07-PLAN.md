<objective>
Surface the grounded draft on the ticket page behind an absolute human gate. An agent clicks "Generate draft" (disabled/softened when the org has no embedded KB), sees a draft card with the reply preview + inline [N] citations linking to KB articles (or an explicit "no relevant sources" state), clicks Insert to load it into the existing Composer for edit/approval, then Send â€” which goes through the EXISTING messages POST route (nothing autonomous). That draft-originated send records a `DRAFT_APPROVED` audit event linked to the sent message. `DRAFT_GENERATED` was already audited by 05-04; this plan closes the audit loop with the approval/send half (Success Criterion 3).

Purpose: This completes AIDA-16's user-facing flow and the human-in-the-loop guarantee (Decision 6). Depends on 05-04's `generateDraftReply` Server Action + the widened AuditActionType from 05-01.
Output: draft card + citation list + a reply-area wrapper coordinating draft->Composer + the DRAFT_APPROVED audit hook in the send route.
</objective>
