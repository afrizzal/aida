<objective>
Turn a saved KB article into stored, searchable vectors. Add a heading-based Markdown chunker (using the already-installed remark parser's AST position offsets — no new dependency), a single `createKbArticle`/`updateKbArticle` write path that renders sanitized HTML and enqueues embedding after commit, and a `kb-embed-article` pg-boss job that chunks + batch-embeds + writes `KbChunk` rows via raw SQL (the only way to write the `Unsupported` vector column). This mirrors the Phase 4 `ai-triage` job shape exactly (kill-switch re-check, retry/backoff, worker + app-side queue registration).

Purpose: This is the "chunked, embedded, and stored in pgvector" half of AIDA-15. 05-05 (Settings re-embed button) and 05-06 (KB authoring UI) both drive this pipeline; 05-04's retrieval reads the KbChunk rows it writes.
Output: chunker + KB write path + embed worker job + queue registration, proven by an integration test with a mocked embed provider.
</objective>
